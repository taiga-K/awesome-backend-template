# プルリクエスト作成（PULL_REQUEST_TEMPLATE 使用）

gh コマンドで `.github/PULL_REQUEST_TEMPLATE.md` を使って **非対話で** プルリクエストを作成する。GitFlow に則り、ブランチ種別に応じた正しいベースブランチを設定する。

## 方針

- 常に非対話で実行する。`--title` と `--body` を必ず指定する
- 現在のブランチをヘッドとして PR を作成する
- ブランチ名から種別を判定し、GitFlow に従ってベースブランチを決定する

## 前提条件

- GitHub CLI (`gh`) がインストール済み
- リポジトリで認証済み (`gh auth status`)
- `.github/PULL_REQUEST_TEMPLATE.md` が存在する
- 作成対象のブランチにコミットがあり、必要なら push 済みであること

## GitFlow によるベースブランチ

| ブランチ種別 | ベースブランチ | 説明 |
|-------------|----------------|------|
| `feature/*` | `develop` | 新機能は develop にマージ |
| `release/*` | `main` | リリース準備完了後、main にマージして本番リリース |
| `hotfix/*` | `main` | 本番の緊急修正は main にマージ |

- `main` が存在しない場合は `master` を使用する
- 上記以外のブランチの場合は `develop` をデフォルトとする

## プルリクエストのタイトル形式

タイトルは **`{type}: {subject}`** の形式とする。

| type | 説明 | 優先度 |
|------|------|------|
| `feat` | 新機能の追加 | 1（最上位） |
| `mod` | 機能変更（仕様変更） | 2 |
| `fix` | バグの修正 | 3 |
| `refactor` | リファクタリング | 4 |
| `docs` | ドキュメントの更新 | 5 |

複数の type に該当する場合は、より上位のものを選ぶ。

## 非対話での作成フロー

1. **テンプレートを読む**: `.github/PULL_REQUEST_TEMPLATE.md` を読み、各項目の意味を把握する
2. **タイトルを決定**: 変更内容から type を選び、`{type}: {subject}` 形式でタイトルを作成する
3. **本文を埋める**: テンプレートの項目をユーザー要件・会話内容・変更差分に基づいて埋める
4. **ベースブランチを決定**: 現在ブランチ名から GitFlow に従って `--base` を設定
5. **コマンドを実行**: `gh pr create` に `--title`、`--body`、`--base` を指定して実行

## テンプレート構造（PULL_REQUEST_TEMPLATE）

タイトルは `{type}: {subject}` 形式で別途指定。本文は以下の項目を含む：

| 項目 | 説明 |
|------|------|
| 概要 | 変更内容の概要 |
| テストと開発プロセス | チェックリスト（テスト追加・テストパス確認） |
| 動機と解決する問題 | 変更の動機と解決する問題 |
| テスト方法 | 変更をテストした方法 |
| 影響範囲 | 修正による影響 |
| レビュー観点 | 見てほしい観点（任意） |
| issue番号 | `closes #123` 形式で記入（あれば） |

## 基本コマンド

```bash
gh pr create --title "<タイトル>" --body "<本文>" --base <ベースブランチ>
```

## よく使うオプション

| オプション | 説明 |
|-----------|------|
| `--title` | タイトル（必須） |
| `--body` | 本文（必須。テンプレートを埋めた内容） |
| `--base` | マージ先（ベース）ブランチ（GitFlow に従う） |
| `--draft` | 下書き PR として作成 |
| `--fill` | ブランチ名やコミットからタイトル・本文を推測（本スキルでは使わず、明示的に指定） |

## 実行例

**feature ブランチから develop への PR:**

```bash
gh pr create --title "feat: ログイン機能の追加" \
  --base develop \
  --body "## 概要
JWT ベースのログイン機能を追加しました。

## テストと開発プロセス
- [x] バグ修正や新しいコードにはテストを追加した
- [x] テストスイートがパスすることを確認した

## プルリクエストの詳細

### 動機と解決する問題
ユーザー認証が必要なため、ログイン機能を実装しました。

### テスト方法
\`npm test\` でユニットテストを実行し、手動でログイン画面の動作確認を実施しました。

## 影響範囲
- 認証ミドルウェアの追加
- ログイン画面の新規作成

## レビュー観点
認証フローのセキュリティ観点で確認をお願いします。

## issue番号
closes #42"
```

**release ブランチから main への PR:**

```bash
gh pr create --title "feat: v1.2.0 リリース" \
  --base main \
  --body "## 概要
バージョン 1.2.0 のリリース準備が完了しました。

## テストと開発プロセス
- [x] バグ修正や新しいコードにはテストを追加した
- [x] テストスイートがパスすることを確認した

## プルリクエストの詳細

### 動機と解決する問題
v1.2.0 で計画された機能のリリースのため。

### テスト方法
CI で全テストがパスすることを確認し、本番相当環境で動作確認しました。

## 影響範囲
v1.2.0 に含まれる全機能。バージョン番号の更新を含む。

## レビュー観点
リリースノートの記載漏れがないか確認をお願いします。

## issue番号
closes #50"
```

**hotfix ブランチから main への PR:**

```bash
gh pr create --title "fix: 決済エラーの緊急修正" \
  --base main \
  --body "## 概要
Null 参照により発生していた決済エラーを修正しました。

## テストと開発プロセス
- [x] バグ修正や新しいコードにはテストを追加した
- [x] テストスイートがパスすることを確認した

## プルリクエストの詳細

### 動機と解決する問題
本番環境で決済処理時にクラッシュする事象を解消するため。

### テスト方法
該当箇所のユニットテストを追加し、既存テスト含めて全件パスを確認しました。

## 影響範囲
決済サービス内の Null チェック追加のみ。他機能への影響はなし。

## レビュー観点
特になし。

## issue番号
closes #99"
```

## 注意事項

- `--title` と `--body` を必ず指定し、エディタ起動を避ける
- 本文に改行・特殊文字を含める場合、シェルでは `\` で継続するかヒアドキュメントを使う
- ユーザーが明示的にベースブランチを指定した場合はその指定に従う
- リモートに push されていない場合、`gh pr create` 実行時に push を促すメッセージが出る場合がある。その場合は先に `git push -u origin <ブランチ名>` を実行する
