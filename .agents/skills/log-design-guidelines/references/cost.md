# 費用

Google Cloud Logging は GB 単位課金・保持期間別ストレージ費用・検索クエリのスキャン課金がかかる。

## ログ量の制御

### 発生源でログを絞る

- ログレベルを適切に設定する
- 用途不明なログは出力しない
- 巨大なペイロード（Base64化バイナリ等）は出力しない
- ただし、調査に必要なログが欠落すれば本末転倒

### 最初から構造化ロギング（JSON Lines）

- ログ基盤側のパース処理は高コスト
- 最初から JSON Lines で出力することで、Google Cloud Logging のパース処理を最小化する

## 保持期間ポリシー

### ビジネス/コンプライアンス要件に基づく

- 環境別に定義する
  - ローカル環境: 保持不要
  - 開発環境: 1週間程度（短く設定してコスト削減）
  - ステージング: 2週間〜1ヶ月
  - 本番: ビジネス要件に基づく（例: 1年、3年）

### Google Cloud Logging での設定

```
# ログバケットの保持期間設定例
gcloud logging buckets update _Default \
  --location=global \
  --retention-days=30
```

## ストレージ階層化

原則不要。一定期間後に安価なアーカイブストレージへ移動するアプローチは、コストメリットが小さい場合が多い。

## コスト意識のあるログ設計の原則

1. **必要十分なログ量**: 多すぎず少なすぎず
2. **ループ処理でのサンプリング**: マシンガンログを避ける
3. **ヘルスチェックの除外**: 不要なアクセスログを削減
4. **環境別ログレベル**: 開発はDEBUG、本番はINFO
5. **巨大データの排除**: リクエストボディ全体やBase64バイナリをログに含めない
