# 例

## 例1: Web API のアクセスログ設計

**入力**

- ユーザー: `Gin のアクセスログ項目を設計して`

**期待する動き**

- まずアクセスログをミドルウェアで一元化する方針を確認する
- `request.id`、`trace_id`、`url.path`、`http.request.method`、`http.response.status_code`、`http.server.request.duration` を候補に出す
- ヘルスチェック除外と `client.address` の扱いを確認する

## 例2: `slog` の実装修正

**入力**

- ユーザー: `このログ、文字列連結をやめたい`

**期待する動き**

- `message` と可変値を分離する
- `logger.InfoContext(ctx, "...", "order.id", orderID)` のように属性化する
- 併せてキー名が既存規約に合うか確認する

## 例3: エラーログのレビュー

**入力**

- ユーザー: `このエラーログ設計をレビューして`

**期待する動き**

- メッセージが具体的か確認する
- メッセージコード、`trace_id`、例外情報の有無を確認する
- ユーザー向け文言との分離、機密情報漏えい、重複ログを確認する

## 例4: バッチログの整理

**入力**

- ユーザー: `バッチのログが多すぎるので整理したい`

**期待する動き**

- 開始、終了、件数、進捗サンプリングへ整理する
- ループ内の逐次ログを減らす
- 障害調査に必要な ID や集計結果が残るか確認する
