# DevOps、CI/CD、およびコンテナ

- `rule_id`: `codeguard-0-devops-ci-cd-containers`
- 原題: `DevOps, CI/CD, and Containers`

ビルド、パッケージング、デプロイのサプライチェーンを保護する。パイプラインと成果物を守り、コンテナを堅牢化し、必要に応じて仮想パッチとツールチェーンのフラグを使う。

### CI/CD パイプラインのセキュリティ

- リポジトリ: 保護ブランチ、必須レビュー、署名付きコミット。
- シークレット: ハードコードしない。実行時に vault／KMS から取得し、ログではマスクする。
- 最小権限: 短命で分離されたランナーに最小限の権限を与える。
- CI のセキュリティゲート: SAST、SCA、DAST、IaC スキャン。重大な問題でブロックする。
- 依存関係: ロックファイルで固定し、整合性を検証し、プライベートレジストリを使う。
- すべてに署名: コミットと成果物（コンテナ／jar など）に署名し、デプロイ前に検証する。SLSA の出所を採用する。

### Docker とコンテナの堅牢化

- ユーザ: 非 root で実行する。Dockerfile で `USER` を設定する。
- `--security-opt=no-new-privileges` で権限昇格を防ぐ。
- ケーパビリティ: `--cap-drop all` とし、必要なものだけ追加する。`--privileged` は使わない。
- デーモンソケット: `/var/run/docker.sock` をマウントしない。
- TLS なしで TCP の Docker デーモンソケット（`-H tcp://0.0.0.0:XXX`）を有効にしない。
- docker-compose で `- "/var/run/docker.sock:/var/run/docker.sock"` を避ける。
- ファイルシステム: root を読み取り専用、一時書き込みは tmpfs。CPU／メモリなどのリソース上限。
- ネットワーク: host ネットワークは避け、カスタムネットワークを定義し、公開ポートを制限する。
- イメージ: 最小ベース（distroless／alpine）、タグとダイジェストを固定。最終イメージからパッケージマネージャとツールを削除する。`HEALTHCHECK` を追加する。
- シークレット: Docker／Kubernetes のシークレット。レイヤーや環境変数に置かない。実行時シークレットでマウントする。
- スキャン: ビルド時と准入時にイメージをスキャンし、深刻な脆弱性でブロックする。

### コンテナ内の Node.js

- 再現可能なビルド: `npm ci --omit=dev`。ベースイメージはダイジェストで固定する。
- 本番環境: `ENV NODE_ENV=production`。
- 非 root: 適切な所有者でコピーし、`USER node` に落とす。
- シグナル: init（例: `dumb-init`）を使い、graceful shutdown ハンドラを実装する。
- マルチステージビルド: ビルドとランタイムを分離する。BuildKit でシークレットをマウントする。`.dockerignore` を使う。

### 仮想パッチ（一時的な緩和）

- コード修正がまだできないときの即時保護に WAF／IPS／ModSecurity を使う。
- 精度のためポジティブセキュリティ（許可リスト）を優先し、特定のエクスプロイト署名だけに頼らない。
- プロセス: 事前にツールを用意する。CVE を分析する。まずログのみで実装し、その後強制する。コード修正後に追跡して廃止する。

### C/C++ ツールチェーンの堅牢化（該当する場合）

- コンパイラ: `-Wall -Wextra -Wconversion`、`-fstack-protector-all`、PIE（`-fPIE`/`-pie`）、`_FORTIFY_SOURCE=2`、CFI（LTO 付き `-fsanitize=cfi`）。
- リンカ: RELRO／now、noexecstack、NX／DEP と ASLR。
- デバッグとリリース: デバッグではサニタイザを有効化。リリースでは堅牢化フラグを有効化。assert はデバッグのみ。
- CI チェック: フラグを検証（`checksec`）し、保護が欠けていればビルドを失敗させる。

### 実装チェックリスト

- パイプライン: シークレットは vault。短命ランナー。セキュリティスキャン。出所付きで成果物に署名。
- コンテナ: 非 root、最小権限、読み取り専用 FS、リソース上限。デーモンソケットのマウントなし。
- イメージ: 最小・固定・スキャン済み。ヘルスチェック。`.dockerignore` の維持。
- Node イメージ: `npm ci`、`NODE_ENV=production`、適切な init とシャットダウン。
- 仮想パッチ: 手順の定義。正確なルール。ログ。修正後の廃止。
- ネイティブビルド: 堅牢化フラグを CI で有効化し検証する。
