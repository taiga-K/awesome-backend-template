# クラウドとオーケストレーション（Kubernetes）

- `rule_id`: `codeguard-0-cloud-orchestration-kubernetes`
- 原題: `Cloud & Orchestration (Kubernetes)`

Kubernetes クラスタとワークロードの堅牢化: アイデンティティ、ポリシー、ネットワーク、シークレット、サプライチェーンの制御。

### コントロール

- アイデンティティと RBAC: ユーザとサービスアカウントに最小権限。名前空間を分離し、必要なロールのみをバインドする。
- ポリシー: イメージの出所、ケーパビリティ、root、ネットワークポリシー、必須ラベル／注釈のための准入制御（OPA／Gatekeeper／Kyverno）。
- ネットワーク: ネットワークポリシーでデフォルト拒否。明示的な egress 許可リスト。該当する場合はメッシュ内のサービスアイデンティティ／mTLS。
- シークレット: KMS プロバイダを使う。マニフェストに平文を置かない。定期的にローテーションし、マウントパスを制限する。
- ノード: OS の堅牢化、自動更新、攻撃面の最小化。taints／tolerations と専用ノードで機密ワークロードを分離する。
- サプライチェーン: イメージ署名を検証する。准入で出所（SLSA／Sigstore）を強制する。

### チェックリスト

- チーム／アプリごとの名前空間。スコープされた RBAC ロール。監査ログの有効化。
- 准入ポリシーでイメージの出所、非 root、ドロップしたケーパビリティ、読み取り専用 root FS、ネットワークポリシの存在を強制する。
- ingress／egress 用のネットワークポリシ。デプロイごとにスコープされたサービスアカウント。

### 検証

- クラスタ適合性と CIS ベンチマークのスキャン。
- CI でのマニフェストのポリシーテスト（OPA 単体テスト）。定期的な准入 dry-run。

### インシデント対応の準備

- 監査ログを有効化し一元化する。etcd へのアクセスを制限する。バックアップ／リストアをテストする。
- MFA と時間制限付き承認を伴う break-glass ロールを定義する。
