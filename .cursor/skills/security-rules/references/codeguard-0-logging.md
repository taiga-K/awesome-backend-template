# ロギングと監視

- `rule_id`: `codeguard-0-logging`
- 原題: `Logging & Monitoring`

検出、対応、フォレンジックを支援しつつ、シークレットを露出させない、構造化されプライバシーに配慮したテレメトリを出力すること。

### 何をログに残すか
- 認証・認可イベント。管理者操作。設定変更。機密データへのアクセス。入力検証の失敗。セキュリティエラー。
- 相関／リクエスト ID、ユーザー／セッション ID（非 PII）、送信元 IP、ユーザーエージェント、タイムスタンプ（UTC、RFC3339）を含める。

### どのようにログに残すか
- 重要なシグナルは自由形式テキストを避け、安定したフィールド名を持つ構造化ログ（JSON）。
- ログインジェクションを防ぐため、すべてのログ入力をサニタイズする（CR／LF／区切り文字を除去する）。他の信頼ゾーンからのデータは検証する。
- シークレットや機微フィールドはマスクまたはトークン化する。クレデンシャル、トークン、リカバリコード、生のセッション ID をログに残してはならない。
- 整合性を確保する：追記のみまたは WORM ストレージ。改ざん検知。集中集約。アクセス制御と保持ポリシー。

### 検知とアラート
- 認証の異常（クレデンシャルスタッフィングのパターン、あり得ない移動）、権限変更、過剰な失敗、SSRF の兆候、データ流出パターンに対するアラートを構築する。
- しきい値を調整する。ランブックを用意する。オンコール体制を確保する。アラートフローをテストする。

### 保管と保護
- ログストレージを分離する（別パーティション／データベース）。厳格なファイル／ディレクトリ権限。Web から到達可能な場所に置かない。
- システム間で時刻を同期する。転送には安全なプロトコルを用いる。改ざん検知と監視を実装する。

### プライバシーとコンプライアンス
- データの目録と分類を維持する。ログ内の個人データを最小化する。保持と削除のポリシーを遵守する。
- ポリシーで求められる場合、ユーザーに紐づくログデータを追跡・削除する仕組みを提供する。

### 実装チェックリスト
- JSON ロギング有効。ログインジェクションのサニタイズ有効。マスクフィルタ有効。すべてのリクエストに相関 ID。
- 改ざん検知付きの分離ログストレージ。整合性保護付きの集中ログパイプライン。保持の設定。
- セキュリティアラートの定義とテスト。ダッシュボードとレポートの整備。

### 検証
- 主要フィールドの有無を検証する単体／統合テスト。マスクの単体テスト。
- シークレット／PII 漏えいの定期監査。インシデントワークフローのテーブルトップ演習。
