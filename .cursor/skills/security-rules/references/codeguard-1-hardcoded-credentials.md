# 資格情報のハードコード禁止

- `rule_id`: `codeguard-1-hardcoded-credentials`
- 原題: No Hardcoded Credentials

秘密、パスワード、API キー、トークン、その他の資格情報をソースコードに直接保存してはならない。

コードベースは公開されており信頼できないものとして扱う。ソースコードに現れる資格情報は漏洩したものとみなし、安全な代替手段で扱わなければならない。

#### 次の種類の値をハードコードしてはならない:

パスワードと認証:
- データベースのパスワード、ユーザーのパスワード、管理者パスワード
- API キー、秘密鍵、アクセストークン、リフレッシュトークン
- 秘密鍵、証明書、署名鍵
- 資格情報を含む接続文字列
- OAuth クライアントシークレット、Webhook の秘密
- 外部サービスにアクセスするのに使えるその他の資格情報


#### 認識パターン — 次の形式に慣れること

ハードコードしてはならない一般的な秘密の形式:

- AWS キー: `AKIA`, `AGPA`, `AIDA`, `AROA`, `AIPA`, `ANPA`, `ANVA`, `ASIA` で始まる
- Stripe キー: `sk_live_`, `pk_live_`, `sk_test_`, `pk_test_` で始まる
- Google API: `AIza` の後に 35 文字
- GitHub トークン: `ghp_`, `gho_`, `ghu_`, `ghs_`, `ghr_` で始まる
- JWT トークン: ドットで区切られた 3 つの Base64 セクション、`eyJ` で始まる
- 秘密鍵ブロック: `-----BEGIN` と `-----END PRIVATE KEY-----` の間の任意のテキスト
- 接続文字列: `mongodb://user:pass@host` のような資格情報付き URL

コードにおける警告サイン:
- `password`, `secret`, `key`, `token`, `auth` を含む変数名
- 何であるか明確でない長いランダムっぽい文字列
- 認証コード付近の Base64 エンコード文字列
- 外部サービスへのアクセスを許可する任意の文字列

このルールをどのように適用したか、なぜ適用したかを常に説明すること。
