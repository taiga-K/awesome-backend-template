# ドキュメント管理

## メッセージ定義書

WARN以上のログは運用手順書と紐づける。メッセージコードに一意な運用フローが定義されるようにする。

### メッセージ定義書の項目

| 分類 | 項目 | 必須 | 内容 | 例 |
|------|------|------|------|-----|
| 出力項目 | メッセージコード | ✅ | 一意なID | AUTH-LOGIN-001 |
| | メッセージ | ✅ | 何が起きたか | ユーザー認証の失敗 |
| | 可変値 | | 可変値 | userID, itemID, ColorCD, SizeCD |
| | エンドユーザー向けメッセージ | | あれば | 「ユーザーIDまたはパスワードを確認してください」 |
| | ログレベル | ✅ | ERROR, WARN, INFO | WARN |
| | 通知フラグ | | 制御したい場合 | true / false |
| 運用定義 | 発生原因 | ✅ | 主な原因・発生条件 | 1. パスワード入力ミス 2. 存在しないユーザーID |
| | アクション | ✅ | 運用者の対応・調査 | IP確認、ブルートフォース攻撃を疑う |

### メッセージコードの命名規則例

```
{機能}-{レベル}-{連番}

例:
  AUTH-E-001    認証系エラー
  AUTH-W-001    認証系警告
  PAYMENT-E-001 決済系エラー
  BATCH-W-001   バッチ系警告
  BIZ-W-1001    ビジネスロジック警告
```

## ログ項目ライフサイクル

暗黙的なログ項目と明示的なログ項目のライフサイクルを一覧表として管理する。

| 出力場所 | トリガー | 項目種別 | 項目例 | 補足説明 |
|----------|----------|----------|--------|-----------|
| フレームワーク | リクエスト単位 | HTTPリクエスト | ステータス、パス、処理時間、リクエスト元IP | Webフレームワークが自動記録 |
| | | DBアクセス | SQLクエリなど | |
| | 例外発生時 | 例外 | メッセージ、レベル | フレームワークでレベル決定 |
| アプリケーション | アプリ呼び出し | ロガー出力項目 | ファイル名、行番号など | slogが自動取得 |
| | | コンテキスト設定項目 | ユーザーID、トレースID | context.Contextで後続処理に引き継ぎ |
| | | 暗黙的設定項目 | パッケージ名 | ロガー初期化時に指定 |
| | | 明示的出力項目 | エラーメッセージ、ステータス、パラメータ | 開発者が実装 |
| | ログレベル指定 | ログレベル | slog.Error()のように指定 | |

## 運用手順書との連携

1. メッセージコードは WARN 以上のログに必ず付与する
2. メッセージコードから運用手順書を一意に引ける構造にする
3. 運用手順書には発生原因と対応アクションを明記する
4. 定期的にメッセージ定義書と実装の整合性を確認する
